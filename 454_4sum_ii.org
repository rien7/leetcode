#+title: 454. 4Sum II
#+filetags: :Medium:

* [[id:2b2e3e1f-924b-43fe-a7f8-a1eb6482d2f2][问题]]
Given four integer arrays ~nums1~, ~nums2~, ~nums3~, and ~nums4~ all of length ~n~, return the number of tuples ~(i, j, k, l)~ such that:

- ~0 <= i, j, k, l < n~
- ~nums1[i] + nums2[j] + nums3[k] + nums4[l] == 0~

*Example 1:*

#+BEGIN_SRC
Input: nums1 = [1,2], nums2 = [-2,-1], nums3 = [-1,2], nums4 = [0,2]
Output: 2
Explanation:
The two tuples are:
1. (0, 0, 0, 1) -> nums1[0] + nums2[0] + nums3[0] + nums4[1] = 1 + (-2) + (-1) + 2 = 0
2. (1, 1, 0, 0) -> nums1[1] + nums2[1] + nums3[0] + nums4[0] = 2 + (-1) + (-1) + 0 = 0
#+END_SRC

Example 2:

#+BEGIN_SRC
Input: nums1 = [0], nums2 = [0], nums3 = [0], nums4 = [0]
Output: 1
#+END_SRC

*Constraints:*

- ~n == nums1.length~
- ~n == nums2.length~
- ~n == nums3.length~
- ~n == nums4.length~
- ~1 <= n <= 200~
- ~-2^{28} <= nums1[i], nums2[i], nums3[i], nums4[i] <= 2^{28}~

* 思路
这题的解题思路十分出乎意料, 众所周知, 这题可以暴力 O(n^4) 解题, 我感觉则其中可以进行一定的优化, 原因是可能有一些中间结果相加的和一致, 可能可以使用动态规划解题.

但是别人的解题思路则是把 O(n^4) 优化成 O(n^2), 其实就是分开两步来做, 既然要四个数相加为 0, 那么任意两个数的和 (a+b) 必然和另外两个数的和 (c+d) 的和为0.

选用任意"两"个数只是为了在最大程度上减少复杂度, 并不意味着不能用 3+1 的方式. 同时, 选用哪两个数字也没有作出限制, 可以随意选择.

值得一提的是, map 中可以初始化大小为 n^2 来用空间换时间

* 代码
#+BEGIN_SRC go
func fourSumCount(nums1 []int, nums2 []int, nums3 []int, nums4 []int) int {
    m := make(map[int]int, len(nums1)*len(nums2))
    res := 0

    for _, n1 := range nums1 {
        for _, n2 := range nums2 {
            m[n1+n2]++
        }
    }

    for _, n3 := range nums3 {
        for _, n4 := range nums4 {
            res += m[0-n3-n4]
        }
    }

    return res
}
#+END_SRC
